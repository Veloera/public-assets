详细设置指南

按照以下步骤快速配置您的开发环境

1

获取 API 令牌

首先需要创建您的专属 API 访问令牌：

warning 重要提示：请勿选择 default 分组！该分组无法正常使用。 选择其他可用的模型分组来创建令牌。

根据您的使用场景选择合适的模型分组：

  • 标准使用：选择常规模型分组
  • Claude Code 之外使用：选择 1.5x 倍率的 "想用其他模型选我" 分组
2

环境配置

根据您的需求选择合适的配置方式：

方案一：临时配置（推荐测试使用）

在每次运行 claude 命令前，先设置环境变量：

export ANTHROPIC_AUTH_TOKEN="您的API令牌"
export ANTHROPIC_BASE_URL="https://becode.be-a.dev"
lightbulb 这种方式只在当前终端会话有效，重启终端后需要重新设置。

方案二：永久配置（推荐生产环境）

编辑配置文件，设置永久环境变量：

# 编辑配置文件（如果不存在请创建）
nano ~/.claude/settings.json
# 文件内容如下：
{ "env": { "ANTHROPIC_AUTH_TOKEN": "您的API令牌", "ANTHROPIC_BASE_URL": "https://becode.be-a.dev" } }
lightbulb 保存文件后，所有 claude 命令都会自动使用这些配置，无需每次手动设置。
3

验证配置

配置完成后，运行测试命令验证是否设置成功：

# 测试连接
claude "你好"
# 或者使用版本检查
claude --version
warning 重要提醒：请勿使用 Claude Code Router 或类似软件！
我们的服务提供原生 Claude Code 支持，直接使用我们的 API 端点即可。

如果遇到问题，请检查：

  • API 令牌是否正确
  • 配置文件路径是否为 ~/.claude/settings.json
  • 网络连接是否正常
  • 是否选择了正确的模型分组
